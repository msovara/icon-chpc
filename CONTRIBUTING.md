# Contributing to ICON Usage Repository

Thank you for your interest in contributing to the ICON usage documentation and templates!

## How to Contribute

### Reporting Issues

If you find errors, unclear documentation, or have suggestions:

1. Open an issue on GitHub describing the problem or suggestion
2. Include:
   - Description of the issue
   - Steps to reproduce (if applicable)
   - Expected vs. actual behavior
   - ICON version and system information

### Contributing Code/Configurations

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Commit with clear messages (`git commit -m 'Add amazing feature'`)
5. Push to your branch (`git push origin feature/amazing-feature`)
6. Open a Pull Request

### Contributing Documentation

1. Follow the existing documentation style
2. Use clear, concise language
3. Include examples where helpful
4. Test any code examples or commands

### Configuration Templates

When adding new configuration templates:

- Include comments explaining each parameter
- Provide reasonable default values
- Document any dependencies or requirements
- Include usage examples

### Code of Conduct

- Be respectful and inclusive
- Provide constructive feedback
- Help others learn and improve

## Development Setup

```bash
# Clone the repository
git clone https://github.com/yourusername/icon-usage.git
cd icon-usage

# Create a branch for your changes
git checkout -b your-feature-branch

# Make changes and commit
git add .
git commit -m "Description of changes"

# Push and create pull request
git push origin your-feature-branch
```

## Style Guidelines

### Configuration Files (.nml)

- Use descriptive comments
- Group related parameters
- Include units where applicable
- Provide example values

### Scripts

- Include header comments
- Use clear variable names
- Add error checking
- Include usage instructions

### Documentation

- Use Markdown formatting
- Include code blocks with syntax highlighting
- Provide practical examples
- Keep language clear and accessible

Thank you for contributing!
